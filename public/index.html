After setting up your authenticator application use the code generated by your authenticator in the following input to test authentication.
This is a service used to test and/or demonstrate TOTP authentication.

A secret is shared between the service and the user's authenticator application along with details on how the secret should be used to generate authentication codes. I.E. the cryptographic hashing algorithm, the number of digits expected in the authentication code, the interval at which new codes are generated and expired.

The current date and time are used with the authentication details to generate codes within the service and the user's device that is running the authenticator application. As long as the server and user device have an accureate date, time, and authentication details they will both generate the same code. The user sends this code to the server to have the service to authenticate the user that is sending the code.

If there is a mis-match between the authentication details or the date and time then the codes generated by the user's authenticator will not match the server and authentication will fail.
